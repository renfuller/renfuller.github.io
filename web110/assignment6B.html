For this week's Wikiversity, I completed the Mozilla's: What is Accessibility and tried using the Paletton website for the first time. The article explained how important accessibility is in web design and how it is often overlooked. It covered things like using good color contrast, creating alternative color schemes for people with different types of color blindness or vision impairments, using icons and graphics to improve understanding, choosing clear and easy-to-read text, and supporting features like text-to-speech. Overall, it showed how these design choices make websites easier for everyone to use. This is especially important to me because I'm interested in a career in UI/UX design, where creating interfaces for websites and games that are easy to understand and accessible to all users is a critical part of the design process.
